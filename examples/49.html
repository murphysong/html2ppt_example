4.2 基础设施逻辑

训练与推理算力：基础设施逻辑的本质分野

训练场景：集中力量办大事
核心逻辑 极致算力密度 + 强同步集群
核心指标 MFU (30%-40%)、线性加速比 (夸娥：91%)
硬件选择 H200 GPU 算力达 1.979EFLOPS，144GB HBM3e
网络架构 NVLink / InfiniBand (400Gbps 单链路)
成本结构 一次性资本投入 (CapEx)，千万美元级单次训练

本质：为了在最短时间内完成万亿参数收敛，允许一定的硬件闲置率，追求梯度数据的实时同步，基础设施呈现高度集中化特征。

推理场景：细水长流保稳定
核心逻辑 分布式协同 + 异构算力兼容
核心指标 Token 吞吐率、端到端时延、能效比
硬件选择 AWS Trainium3 推理吞吐为训练 3 倍，能效比 +40%
网络架构 全光互联 / RDMA (时延 < 3微秒，华为灵衢)
成本结构 持续性运营成本 (OpEx)，支出占比由 30% 飙至 70%

本质：为了响应百万级并发请求，算力必须分布化与下沉。本地边缘节点延迟（45ms）比云端（1200ms）降低 26 倍，决定商业存续。
